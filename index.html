Module 1

Why Bother? UX & UDL

Understanding the frustration of bad design before building the solution.

Module Overview

Before we write code, we must understand the "why." We will explore how User Experience (UX) impacts teacher wellbeing and how Universal Design for Learning (UDL) applies to digital staffrooms.

Success Criteria

  • Define UX failures in school portals.
  • Explain the "3-Click Rule".
  • Critique resources using the 3 Pillars of UDL.

Activity: See-Think-Wonder

Discussion

Why are we doing this? Before we can build better tools, we must learn to identify bad design objectively. This routine separates observation from judgment.

LO 2.1: Critique UX Design LO 2.2: Apply Critical Inquiry
Example Response:
"See: I see a staff portal with 12 unlabelled folder icons.
Think: I think this would make a new teacher feel anxious about where to save files.
Wonder: I wonder if color-coding them by department would solve this?"
Criteria Developing (1pt) Proficient (2pts)
Observation Vague description ("It looks bad"). Specific, objective detail ("The font size is 8px").
Analysis States a feeling without cause. Connects design feature to user emotion/outcome.
Inquiry No question posed. Asks a "How might we..." question for improvement.
